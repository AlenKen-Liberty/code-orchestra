# 简易计算器 Python 函数 — 设计方案

## 1. 系统架构概览

```
┌─────────────────────────────────────┐
│           calculator()              │
│  ┌───────────┐  ┌────────────────┐  │
│  │ 输入验证   │→│  运算分发器     │  │
│  │ (validate) │  │ (dispatch)     │  │
│  └───────────┘  └───────┬────────┘  │
│                         │           │
│          ┌──────────────┼────────┐  │
│          ▼      ▼       ▼        ▼  │
│        add    subtract  multiply divide │
└─────────────────────────────────────┘
```

**模块说明：**

| 模块 | 职责 |
|------|------|
| `calculator(a, op, b)` | 主入口函数，接收两个操作数和一个运算符 |
| 输入验证 | 校验操作数类型和运算符合法性 |
| 运算分发器 | 根据运算符映射到对应算术运算 |

## 2. 核心设计决策与理由

| 决策 | 理由 |
|------|------|
| **单函数入口** | 任务为"simple calculator function"，无需类或模块拆分 |
| **字典分发替代 if-elif 链** | 可扩展、更 Pythonic、O(1) 查找 |
| **仅支持二元运算** | 符合简易计算器定位，降低复杂度 |
| **抛异常而非返回错误码** | Python 惯例，调用方可自行 try/except |

## 3. 技术栈选择

| 项目 | 选择 | 理由 |
|------|------|------|
| 语言 | Python 3.x | 任务要求 |
| 依赖 | 无第三方依赖 | 纯算术运算，标准库足够 |
| 测试 | `pytest` 或内置 `unittest` | 轻量验证 |

## 4. 实现步骤

### Phase 1：核心实现

1. 定义 `calculator(a, operator, b)` 函数
2. 用字典映射四则运算：`+`, `-`, `*`, `/`
3. 添加输入验证（类型检查、除零检查、非法运算符）

### Phase 2：测试

4. 编写单元测试覆盖：正常运算、除零、非法输入、边界值（大数、浮点精度）

### Phase 3：可选扩展

5. 如需要，增加取模 `%`、幂运算 `**` 等

## 5. 潜在风险与 Mitigation

| 风险 | 影响 | Mitigation |
|------|------|------------|
| 除零错误 | 程序崩溃 | 显式检查，抛出 `ZeroDivisionError` 并附清晰信息 |
| 非数值输入 | TypeError | 入口处类型校验，拒绝非 `int`/`float` |
| 浮点精度问题 | `0.1 + 0.2 ≠ 0.3` | 文档说明；如需精确计算可引入 `decimal` 模块 |
| 非法运算符 | 静默失败 | 不在映射表中则抛 `ValueError` |
